# SIEM Setup Guide — Ingesting Linux Logs

## Overview
This document describes how to ingest the parsed log data generated by 
this project into a SIEM platform for visualisation and alerting. 
Two options are covered — Splunk Free and Microsoft Sentinel.

---

## Option 1 — Splunk Free

### Prerequisites
- Splunk Free installed (splunk.com/en_us/download.html)
- `output/parsed_logs.csv` generated by log_parser.py

### Step 1 — Start Splunk
```bash
sudo /opt/splunk/bin/splunk start
```
Access Splunk at `http://localhost:8000`

### Step 2 — Upload CSV Data
1. Click **Settings → Add Data**
2. Select **Upload**
3. Choose `output/parsed_logs.csv`
4. Set Source Type to `csv`
5. Click **Next → Review → Submit**

### Step 3 — Search the Data
Once uploaded run these searches in the Search bar:

**All failed logins:**
source="parsed_logs.csv" event_type="FAILED_LOGIN"
**Top attacking IPs:**
source="parsed_logs.csv" event_type="FAILED_LOGIN"

| stats count by ip

| sort -count

**Suspicious user logins:**
source="parsed_logs.csv" event_type="SUCCESSFUL_LOGIN"

| where user IN ("root","admin","anonymous","hacker","scanner","ftpuser")

**Sudo usage:**
source="parsed_logs.csv" event_type="SUDO_USAGE"

| table timestamp user details

### Step 4 — Build a Dashboard
1. Run each search above
2. Click **Save As → Dashboard Panel**
3. Create a new dashboard called `Linux Auth Monitor`
4. Add each search as a separate panel
5. Set visualisation types — bar chart for IPs, table for sudo usage, pie for event types

### Step 5 — Create an Alert
1. Run the failed login search
2. Click **Save As → Alert**
3. Set trigger condition: number of results greater than 10
4. Set alert action: log event
5. Name it `Brute Force Detection Alert`

## Option 2 — Microsoft Sentinel

### Prerequisites
- Azure free account (azure.microsoft.com/free)
- Log Analytics Workspace created
- Microsoft Sentinel enabled on the workspace

### Step 1 — Create Log Analytics Workspace
1. Go to Azure Portal → Log Analytics Workspaces
2. Click **Create**
3. Resource Group: create new → `siem-project-rg`
4. Name: `linux-log-analysis-workspace`
5. Region: North Europe (Ireland)
6. Click **Review + Create**

### Step 2 — Enable Microsoft Sentinel
1. Go to Microsoft Sentinel in Azure Portal
2. Click **Create**
3. Select your Log Analytics Workspace
4. Click **Add**

### Step 3 — Upload Custom Logs
1. In Sentinel go to **Settings → Workspace Settings**
2. Click **Custom Logs → Add Custom Log**
3. Upload `output/parsed_logs.csv` as a sample
4. Set collection path and name the table `LinuxAuthLogs_CL`
5. Click **Create**

### Step 4 — Query the Data
In Sentinel go to **Logs** and run these KQL queries:

**All failed logins:**
```kql
LinuxAuthLogs_CL
| where event_type_s == "FAILED_LOGIN"
| project timestamp_s, user_s, ip_s
```

**Top attacking IPs:**
```kql
LinuxAuthLogs_CL
| where event_type_s in ("FAILED_LOGIN", "INVALID_USER")
| summarize count() by ip_s
| order by count_ desc
| take 5
```

**Suspicious successful logins:**
```kql
LinuxAuthLogs_CL
| where event_type_s == "SUCCESSFUL_LOGIN"
| where user_s in ("root","admin","anonymous","hacker","scanner","ftpuser")
| project timestamp_s, user_s, ip_s
```

**Sudo privilege escalation:**
```kql
LinuxAuthLogs_CL
| where event_type_s == "SUDO_USAGE"
| project timestamp_s, user_s, details_s
```

### Step 5 — Create Analytics Rule
1. Go to Sentinel → **Analytics**
2. Click **Create → Scheduled Query Rule**
3. Name: `Brute Force Detection`
4. Paste the top attacking IPs query
5. Set threshold: count greater than 10
6. Set frequency: every hour
7. Click **Review + Create**

## Which Option to Use

| | Splunk Free | Microsoft Sentinel |
|---|---|---|
| Cost | Free | Free tier available |
| Setup time | 30 mins | 1 hour |
| Best for | Local testing | Cloud/Enterprise |
| Relevance to Irish jobs | High | Very High |

**Recommendation:** If you're targeting Irish enterprise roles use 
Microsoft Sentinel — Azure dominates the Irish market and Sentinel 
experience is explicitly requested in many SOC job postings.
